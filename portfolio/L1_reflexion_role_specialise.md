## Cas choisi

GitHub Copilot Enterprise

## 1. Probleme d'affaires resolu

Dans une organisation technologique de grande taille (comme Accenture), les développeurs consacrent une part importante de leur temps à des tâches répétitives. Cette situation ralentit la livraison des projets, réduit l’efficacité opérationnelle et limite le temps disponible pour d'autres tâches qui nécessitent plus de stratégie et de créativité. C'est pour régler ce genre de problème que GitHub Copilot Entreprise a été créé.

Source: GitHub. “Accenture Customer Story.” Consulté le 13 mai 2026. https://github.com/customer-stories/accenture

## 2. Fonction d'affaires ciblee

Développement logiciel

## 3. Role specialise que l'agent orchestre

Agent développeur 

## 4. Valeur creee — quantifiee avec donnees publiques

95 % des développeurs aiment davantage coder lorsqu’ils utilisent GitHub Copilot.
67 % des développeurs utilisent GitHub Copilot chaque jour.
96 % de taux de satisfaction chez les utilisateurs de GitHub Copilot.

Source: GitHub. “Accenture Customer Story.” Consulté le 13 mai 2026. https://github.com/customer-stories/accenture

## 5. Risque principal et mitigation concrete

Le principal risque associé à l’utilisation de GitHub Copilot Enterprise est la génération de code erroné, vulnérable ou non conforme aux normes de sécurité de l’organisation.

## 6. Condition de succes pour votre organisation

L’organisation devra également :
1. Établir des politiques concernant l'utilisation de l’IA dans le contexte professionnel;
2. Maintenir en tout temps une supervision humaine ;
3. Assurer la sécurité des données pour éviter la fraude et la sécurité de possibles clients ;
4. Mesurer régulièrement les gains de productivité et l’adoption interne pour savoir comment l'IA aide et comment elle pourrait possiblement aider davantage.
